# 聖境塔羅 (Celestial Tarot)

探索靈魂的指引：以純前端打造的沉浸式塔羅占卜平台。整合 MediaPipe 手勢辨識、Gemini AI 深度解牌、PWA 以及極致華麗的「每日一抽」動態儀式。

目前版本：`v1.7.2` (2026-04-24 每日一抽動畫大改版)

## 核心功能

### 🌟 每日一抽 (Daily Guidance) - 極致儀式感
- **九階段魔幻開牌儀式**：結合星雲呼吸、旋轉魔法陣、塔羅浮遊符文與 Canvas 粒子星塵。
- **動態掃描系統**：由 `requestAnimationFrame` 驅動的掃描光束，執行「左右擺掃 -> 漸慢收斂」的三階段路徑。
- **互動反饋 (Peeking Effect)**：光束掃過時卡牌會隨之微微彈起並發光，營造「與命運接觸」的觸感。
- **無縫接棒過場 (Seamless Morph)**：利用動態座標計算，讓抽出的卡牌從原位滑動並放大至螢幕中央，視覺無間斷。
- **真隨機保證**：後台從完整 78 張牌庫中公平抽選，不受視覺扇形牌陣 position 影響。
- **每日一次限制**：基於 `localStorage` 鎖定今日指引，支援強制重新抽取。

### 🌀 3D 抽牌環 (Classic Reading)
- **景深互動**：扇形卡牌圓環，支援手勢旋轉、鎖定與抽取。
- **自由提問**：AI 會根據您的問題動態定義三張牌的角色（如：優點/缺點/建議），提供深度的星辰指引。
- **視覺回饋**：卡牌翻面飛入槽位，支援正逆位判定。

### 🖐️ 專業級手勢辨識
- 整合 Google MediaPipe 模型，讓您無需觸碰螢幕即可操作：
  - **張開手掌**：旋轉牌環。
  - **握拳**：鎖定卡牌。
  - **食指比 1**：確認選牌。

### ⚡ 智慧效能分級 (Adaptive Performance)
系統會自動偵測裝置硬體（CPU、記憶體）與「減少動作」偏好，動態劃分渲染模式：
- **Full 模式 (PC/高效能裝置)**：全動態背景、150 顆發光粒子、全透明模糊濾鏡。
- **Mobile 模式 (一般手機)**：隱藏 Lens Flare，採用簡化粒子渲染，優化行動端流暢度。
- **Lite 模式 (低階裝置/省電模式)**：關閉 Canvas 與複雜濾鏡，確保核心抽牌邏輯不卡頓。

### 📊 結果匯出與日誌
- **圖片輸出**：一鍵產生包含「提問 + 卡牌組 + AI 解讀」的精美結果圖。
- **占卜日誌**：自動紀錄歷史占卜結果，方便隨時回顧。

## 快速開始

本專案為 Vanilla HTML/CSS/JavaScript 原生開發，無需安裝繁重的建置工具。

1. **啟動建議**：請使用 Web Server 開啟（避免直接雙擊 `.html` 檔案導致路徑或權限問題）：
   ```bash
   # 使用 Python 快速啟動
   python -m http.server 8000
   ```
2. **存取**：開啟 `http://localhost:8000`。
3. **AI 解讀**：點擊右上角設定，填入您的 **Google AI API Key** 即可開啟完整的星辰指引功能。

## 專案結構

- `/js`：核心邏輯目錄。
  - `app.js`：流程控制、每日一抽系統。
  - `ring.js`：3D 牌環渲染與手勢互動。
  - `analysis.js`：AI 解讀邏輯。
  - `imageExport.js`：Canvas 圖片繪製與匯出。
- `/css`：
  - `style.css`：主要樣式表（包含效能分級對應樣式）。
- `/assets`：
  - `images/`：Rider-Waite-Smith 版塔羅牌圖檔。

## 常見問題

- **手勢初始化很慢？**
  第一次執行需要下載約 5MB 的手勢識別模型，之後會自動快取。請確認已同意相機授權。
- **為什麼我只能抽大阿爾克那？**
  當內建 API 載入失敗時，系統會自動轉向 Local 儲存的 22 張主牌，確保基本運作。
- **手機滑動不順暢？**
  系統已自動啟用效能分級，若仍感到卡頓，可以檢查是否開啟了手機的省電模式或「減少動畫」選項。
